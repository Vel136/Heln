Heln

A multi-track animation controller for Roblox rigs.

Heln reads a KeyframeSequence into per-joint timelines once, then evaluates those timelines against a Model's Motor6D joints every frame and writes directly to Motor6D.Transform. This bypasses AnimationController and Animator, so animations do not need to be published as Roblox Animation assets and can run on any rig with matching joint names.

Joints are resolved by matching each Pose's name against Motor6D.Part1.Name on the rig, cached per rig and invalidated when a Motor6D is added or removed. 

local Heln = require(ReplicatedStorage.Heln)

local track = Heln.new(rig, keyframeSequence)
track:Play()

track.Ended:Connect(function()
    print("done")
end)

Play​

</>
Heln:Play(
fadeTime: number?,--

Seconds to blend weight. Defaults to instant.

fadeStyle: string?,--

"Linear", "Constant", "Elastic", "Cubic", or "Bounce". Defaults to "Linear".

weight: number?,--

Target blend weight. Defaults to the current WeightTarget.

speed: number?--

Playback speed multiplier. Negative plays backward.

) → ()

Starts or re-fades the track. If the track is already playing, this re-fades WeightCurrent toward weight instead of restarting. 

track:Play()

-- fade in over 0.3s at half weight, double speed
track:Play(0.3, "Linear", 0.5, 2)

Pause​

</>
Heln:Pause() → ()

Freezes the track at its current TimePosition. Calling Play again resumes from there. 

track:Pause()

Stop​

</>
Heln:Stop(
fadeTime: number?,--

Seconds to fade weight to 0 before stopping. Defaults to instant.

fadeStyle: string?--

Easing curve for the fade. Defaults to "Linear".

) → ()

Stops the track, honoring its PlaybackType. Persistent playback types hold their current pose; reversible playback types begin playing backward toward time 0 or a marker before finishing. 

track:Stop()

-- fade weight to 0 over half a second before stopping
track:Stop(0.5)

GetTimeOfKeyframe​

</>
Heln:GetTimeOfKeyframe(keyframeName: string) → number?

Returns the time of a named keyframe from the sequence, or nil if no keyframe with that name exists. 

local time = track:GetTimeOfKeyframe("Landed")

SetMarker​

</>
Heln:SetMarker(
time: number,
name: string?,--

Defaults to an empty string.

value: any--

Payload passed to KeyframeReached and marker signals.

) → ()

Adds a custom marker at time, independent of any Keyframe instance. Custom markers are searched alongside keyframe-derived markers by GetPreviousMarker, GetNextMarker, and GetNearestMarker, and fire KeyframeReached the same way. 

track:SetMarker(0.4, "HitFrame")

GetPreviousMarker​

</>
Heln:GetPreviousMarker(fromTime: number) → Marker?

Returns the closest marker (keyframe-derived or custom) strictly before fromTime, or nil if there is none. 

local marker = track:GetPreviousMarker(1.0)

GetNextMarker​

</>
Heln:GetNextMarker(fromTime: number) → Marker?

Returns the closest marker strictly after fromTime, or nil if there is none. 

local marker = track:GetNextMarker(1.0)

GetNearestMarker​

</>
Heln:GetNearestMarker(fromTime: number) → Marker?

Returns whichever marker is closest to fromTime, preferring the next marker on a tie. 

local marker = track:GetNearestMarker(1.0)

GetMarkerReachedSignal​

</>
Heln:GetMarkerReachedSignal(name: string) → Signal

Returns a signal scoped to markers with the exact name name, created lazily on first request. 

track:GetMarkerReachedSignal("Footstep"):Connect(function(name, value)
    playFootstepSound(value)
end)

Destroy​

</>
Heln:Destroy() → ()

Stops the track, unregisters it from the rig's update loop, and destroys all of its signals, including any created via GetMarkerReachedSignal. Reading or writing properties on a destroyed track raises an error. 

track:Destroy()
